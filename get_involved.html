Help us build the greatest security library for Java

Hi,

I'm the creator and main developer of pac4j and this is a call for contributors.

pac4j is both:

  • a core library to handle authentication/user profile retrieval for almost all protocols: OAuth, CAS, OpenID, SAML, HTTP, OpenID Connect, GAE and thus for Facebook, Twitter, Google, Yahoo, etc. Most of the logic is here (around 31 000 lines of source code)
  • a set of bridges / implementations (a few classes) to use this core engine in many frameworks/environments: CAS, J2E, Play, Spring Security, Shiro, Ratpack, Vertx, Undertow and Spark.

pac4j is easy. It relies on one core class / concept: the client.

pac4j is consistent across various protocols (while even great ecosystem like Spring offers many different libraries: spring-security-cas, spring-security-saml, Spring Social...)

pac4j libraries are very similar in their use and even in their implementations, whether the frameworks use handlers, filters, routes... Only two filters: one to protect a resource and one to finish the authentication process.

The roadmap ahead is promising: while pac4j was primarily focused on stateful mechanisms, it should be fairly straightforward to address stateless authentications for REST web services, as well as to support authorizations or whatever you may think of.

Maintaining and developing everything is a tremendous job and that's why we are looking for regular committers. To bring pac4j to the next level: to make it the greatest security library for Java!

Why should you get involved?

It's me, Michaël, a bunch of casual contributors and all pac4j users you will help, not a big company trying to make you work for free.

Security is a core issue in applications development, a matter you should be interested in.

We are nice guys ;-)

How to?

The easiest way is to clone the source code (the core and the implementations you may be interested in) and take a look at the latest issues which are the incoming roadmap.

You can also use the mailing lists for questions: pac4j users and pac4j developers.
